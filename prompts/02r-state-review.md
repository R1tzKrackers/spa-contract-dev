# 状態管理レビュー

## 目的
状態管理設計成果物の人間レビューを実施する。

## レビュー対象
- `{{SPA_SPEC_DIR}}/state/`

## レビュー観点
1. **適切性**: 状態がグローバル/ローカルに適切に分離されているか
2. **一貫性**: アクション・セレクタの命名が統一されているか
3. **パフォーマンス**: 不要な再レンダリングを引き起こさないか
4. **テスタビリティ**: テスト可能な設計か

## 判定
- **承認**: 次フェーズ（ルーティング設計）へ進む
- **差し戻し**: 状態管理修正フェーズに戻る

---

## 履歴記録（必須）

フェーズ完了時、`.phase-manager-history.yml` に以下を追記せよ：

```yaml
- phase: "02r-state-review"
  status: "complete"  # または "reject"（差し戻しの場合）
  comment: "レビュー結果を1行で"
  target: "03-state-fix"  # 差し戻し先（rejectの場合のみ）
  timestamp: 現在時刻を取得して記録
```
