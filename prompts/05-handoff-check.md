# 引継ぎ判定

## ロール
{{DESIGN_ROLE}}（設計担当）

## 目的
実装への引継ぎが可能か判定する。

## チェックリスト
- [ ] コンポーネント仕様が完成している
- [ ] 状態管理設計が完了している
- [ ] ルーティングが定義されている
- [ ] データフローが明確である
- [ ] API連携仕様が定義されている
- [ ] エラーハンドリングが定義されている

## 判定
- **承認**: 契約凍結へ進む
- **差し戻し**: 詳細設計フェーズに戻る

---

## 履歴記録（必須）

フェーズ完了時、`.phase-manager-history.yml` に以下を追記せよ：

```yaml
- phase: "05-handoff-check"
  status: "complete"  # または "reject"（差し戻しの場合）
  comment: "判定結果を1行で"
  target: "04-detail-design"  # 差し戻し先（rejectの場合のみ）
  timestamp: 現在時刻を取得して記録
```
